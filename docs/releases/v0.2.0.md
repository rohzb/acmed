# acmed v0.2.0

## Highlights

- Introduced dual issuer execution modes: `local` and `remote`.
- Added remote plugin adapter (`remote_http`) with capabilities/version checks.
- Added remote issuer config fields (`mode`, `remote.endpoint_url`, auth and timeout settings).
- Added normalized `reason_code` propagation in issuance results.
- Consolidated gen2 into a single canonical `acmed` repository with deployment presets in `deploy/`.
- Added deployment compose variants for both source-build and prebuilt-image modes.
- Renamed published runtime image references from `ghcr.io/rohzb/acmed-core:*` to `ghcr.io/rohzb/acmed:*`.

## Notes

- This release preserves compatibility for existing local issuer flows.
- Remote plugin mode is the baseline for gen2 multi-container plugin architecture.
- Migration details for layout and naming changes are documented in `docs/guides/migration-from-gen2-split.md`.
