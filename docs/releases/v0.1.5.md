# acmed v0.1.5 Release Notes

Release date: 2026-04-19

## Summary

This patch release focuses on release-readiness and security posture for public
publishing.

## Highlights

- TLS config is now enforced and implemented at runtime:
  - `server.tls_enabled=true` requires `server.tls_cert_file` and
    `server.tls_key_file`.
  - Runtime wraps the listener socket with TLS (minimum TLS 1.2).
- Docs and examples were hardened for public sharing:
  - Generic example hostnames replaced environment-specific names.
  - Config examples better separate local HTTP testing from TLS production use.
- Test ergonomics improved:
  - `pytest` now runs from repo root without `PYTHONPATH=src`.

## Upgrade Notes

- If you run with `server.tls_enabled=true`, you must provide valid cert and key
  file paths in configuration.
- If you run local development without TLS, keep `server.tls_enabled=false` and
  `server.development_mode=true`.

## Validation

- Full unit test suite passes for this release candidate.
