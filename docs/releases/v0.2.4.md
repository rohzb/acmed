# acmed v0.2.4 Release Notes

Release date: 2026-04-26

## Highlights

- Default behavior for `acme_sh` is now reuse-first (no forced reissue).
- Repeat domain requests that return acme.sh exit code `2` with the
  force-renew hint are treated as reusable-state flows and continue to
  `--install-cert`.

## Operational Impact

- Repeated requests for a currently valid certificate no longer fail with
  `issuer failed with exit code 2`.
- Existing certificate material can be returned through the normal artifact
  pipeline without forcing renewal every time.
