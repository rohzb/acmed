# acmed v0.2.2 Release Notes

Release date: 2026-04-26

## Highlights

- Fixed timeout error propagation in subprocess-based issuer wrappers so
  captured command output is always persisted as text.
- Added worker-level fallback logging for issuer backend exceptions so
  operations can always inspect failure context from artifact logs and audit
  events.

## Operational Impact

- Timeout-related issuer failures now surface as explicit issuer errors with
  artifact log paths, instead of opaque worker crashes.
- ACME order failures caused by backend exceptions now include
  `issuer-output.log` and `challenge-output.log` artifacts for every attempt.
