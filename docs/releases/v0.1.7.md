# acmed v0.1.7 Release Notes

Release date: 2026-04-19

## Summary

This patch release focuses on CI quality gates, repository security checks, and
test-runtime hardening.

## Highlights

- Added workflow coverage for:
  - Python test matrix (`3.11` through `3.14`)
  - Docker build and health-check smoke testing
  - Expanded security checks using Trivy, Gitleaks, and TruffleHog
- Improved release safety by requiring successful `security-checks` status on
  the tagged commit before release publishing.
- Hardened chain test tooling to run as non-root:
  - Test client Docker image now uses a dedicated unprivileged user
  - TLS trust setup now supports non-root environments cleanly

## Upgrade Notes

- No ACME API contract changes.
- No configuration schema changes for runtime deployments.
- CI branch protection rules should target the new check name
  `security-checks`.

## Validation

- Unit test suite passes for this release candidate.
- Release pipeline enforces tag/version/changelog coherence.
- Release pipeline now enforces prior successful security checks.
