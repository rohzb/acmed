# acmed v0.2.1

Release date: 2026-04-26

## Summary

This patch release improves issuer failure observability during order processing.
When issuance fails, logs and surfaced errors now include a concise failure
summary plus direct paths to worker artifact logs.

## Highlights

- Added worker audit event `order.issuance_failed` with issuer + artifact path metadata.
- Failure exceptions now include:
  - last meaningful issuer stderr/stdout line
  - `issuer-output.log` and `challenge-output.log` paths

## Compatibility

- Backward-compatible patch release.
- No API contract or configuration schema changes.

## Validation

- Python syntax validation for updated worker module.
- Existing playbook syntax validation for deployment-side integrations.
